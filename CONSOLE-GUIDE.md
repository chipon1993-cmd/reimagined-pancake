# Command Console — Navigation Panel Guide

## Architecture Overview

`index.html` (the console) is the landing page of the site, styled as a spaceship command console. It serves as the central navigation hub linking to all site modules.

**Key principle:** The console is the entry point. The original homepage content lives at `home.html` and is accessed as one of the modules.

---

## File Structure

```
index.html            ← Console page (landing page, data-page="console")
home.html             ← Original homepage (hero, nav cards, quote, data-page="index")
sw.js                 ← Cache version ac-v5, includes /home.html
loader.js             ← Footer has ◈ link to index.html (console) + ⚙ to admin
admin.html            ← Preview pageMap: index tab → home.html
sitemap.xml           ← Includes /home.html entry
```

### What was NOT modified
- `style.css` — No global CSS changes
- `nav.js` — No navigation logic changes
- `effects.js` — Loaded as-is (page transitions, cursor, particles all work)
- `site-data.js` — No data schema changes
- `translations.js` — No translation entries added
- `about.html`, `journey.html`, `interests.html`, `interest.html`, `videos.html`, `contact.html` — Unchanged (nav logo `index.html` now correctly points to console)

## Console Page Anatomy

### 1. HUD Scanlines (`div.hud-scanlines`)
- Full-screen fixed overlay with subtle repeating gradient lines
- Creates CRT monitor effect
- Adapts color to active theme via `[data-theme]` CSS overrides
- **To adjust intensity:** Change the `rgba` alpha in the `background` property (currently `0.008`)

### 2. Corner Brackets (`div.corner-bracket`)
- 4 decorative elements at screen corners (HUD targeting brackets)
- Hidden on mobile (`display: none` at 768px)
- **To remove:** Delete the 4 `<div class="corner-bracket">` elements

### 3. System Bar (`div.sys-bar`)
- Fixed bar below the main nav (top: 60px)
- Shows: system status, Firebase status, real-time clock, date
- **Firebase indicator:** Checks `window.fsGet` after 2s delay, shows OK/OFFLINE
- **Clock:** Updates every 1 second via `setInterval`
- **To add new indicators:** Add `<div class="sys-indicator">` inside `.sys-bar-group`

### 4. Console Header (`div.console-header`)
- Title "Командная Консоль" with gradient text
- Subtitle with monospace font
- Uses `.fade-in` for scroll animation

### 5. Radar Section (`div.radar-section`)
Two-column layout (flex): radar on left, status panels on right.

**Radar (`div.radar-container`):**
- 320x320px circle with concentric rings and crosshairs
- Sweep arm: `animation: radarSweep 6s linear infinite` (full rotation)
- 5 navigation nodes positioned around the circle (absolute positioning)
- Each node is an `<a>` link to a page
- **To change sweep speed:** Modify `6s` in the `radarSweep` animation
- **To reposition nodes:** Edit the CSS for `.radar-node.n-{name}` (top/left/right/bottom values)

**Status Panel (`div.status-panel`):**
- 3 status blocks showing system state, navigation, and data loading
- Progress bars animate on page load (delayed via setTimeout cascade)
- Theme name, last visited page, session timer are live
- **To add a new status block:** Copy an existing `.status-block` div

### 6. Quick Actions Bar (`div.quickbar`)
- Horizontal button row for fast navigation
- Each button shows emoji icon + label + keyboard shortcut hint
- Keyboard hints hidden on mobile
- **To add a new quick action:** Add `<a class="quick-btn">` element

### 7. Module Cards (`div.modules-grid`)
- 6 cards in responsive grid (min 280px columns)
- Each card: icon, status badge, title, description, tags, arrow
- Admin card has purple accent (inline styles override)
- Top-border glow appears on hover (via `::before` pseudo-element)
- **To add a new module card:** Copy existing `<a class="module-card">` block

### 8. Ship Log (`div.shiplog`)
- Simulated log entries generated by JS on page load
- Shows: timestamp, message, type badge (NAV/SYS/INFO)
- Entries are generated from current time with offsets
- **To add real log entries:** Replace the static `logs` array in JS with data from localStorage or Firestore

### 9. Keyboard Navigation (JS)
- `H` → Home, `1` → About, `2` → Journey, `3` → Interests, `4` → Videos, `5` → Contact
- `ESC` → Back to Home
- Disabled when user is focused in input/textarea
- Uses page transition overlay for smooth navigation
- **To change shortcuts:** Edit the `keyMap` object in the script

### 10. Footer Telemetry (`div.console-footer`)
- Stats: module count, language count, theme count
- Branding labels on left/right

## Theme Compatibility

The console respects all 4 preset themes + custom theme:
- **CSS overrides** at bottom of `<style>` block handle each `[data-theme]`
- Radar sweep, scanlines, borders all adapt color
- All CSS uses `var(--accent)` and `var(--accent2)` where possible
- Theme-specific overrides only exist where CSS variables don't apply (gradient backgrounds in pseudo-elements)

**To add a new theme:**
1. Add `[data-theme="newtheme"]` CSS block at the bottom of the `<style>`
2. Override: `.radar-sweep::after`, `.radar-bg`, `.radar-ring`, `.radar-cross-*`, `.radar-sweep-trail::after`, `.hud-scanlines`, `.sys-bar`

## Safe Editing Rules

1. **Never remove `data-page="console"`** from the `<body>` — effects.js and loader.js use it
2. **Keep the script load order** identical to other pages (lucide → firebase → config → site-data → translations → loader → nav → effects)
3. **Don't add `case 'console'`** to loader.js switch — the page intentionally skips page-specific rendering
4. **Radar node positions** are CSS-based (not calculated) — adjust manually if adding/removing nodes
5. **System bar `top: 60px`** assumes standard nav height — if nav height changes, update this value
6. **Clock/timer intervals** are never cleared — this is fine since they stop when the user navigates away (page unload)
7. **The `ac_last_page` localStorage key** is shared across pages — if you add tracking to other pages, use the same key

## Quick Customization Reference

| What | Where | How |
|------|-------|-----|
| Radar speed | CSS `@keyframes radarSweep` | Change `6s` duration |
| Scanline intensity | CSS `.hud-scanlines` | Change `rgba` alpha (0.008) |
| Module descriptions | HTML `.module-desc` | Edit text directly |
| Keyboard shortcuts | JS `keyMap` object | Add/change key→URL pairs |
| Status indicators | JS init block | Modify checks and values |
| Log messages | JS `logs` array | Change text/types |
| Corner bracket size | CSS `.corner-bracket` | Change width/height (40px) |
| Node positions | CSS `.radar-node.n-*` | Adjust top/left/right/bottom |
