# CLAUDE.md — Инструкции для Claude Code

## Проект
Персональный сайт chepelovskyi.com. Владелец — НЕ программист. Все изменения должны работать через админку (admin.html) без написания кода.

## Перед любой задачей
1. Прочитай PROJECT_ARCHITECTURE.md — там полная схема проекта
2. Проверь что твои изменения не ломают: админку, публичный сайт, Firestore запись/чтение
3. После изменений — протестируй открыв файл в браузере

## Архитектура (кратко)
- Статический HTML на GitHub Pages + Firebase Firestore для данных
- site-data.js = дефолты, loader.js = рендер, admin.html = редактирование
- Данные мержатся: AC_DEFAULTS + Firestore overrides → deepMerge → render

## Правила
- НИКОГДА не добавляй функционал который требует от пользователя писать код или использовать консоль
- ВСЕГДА экранируй пользовательские данные через esc() / safeParagraph() / safeHref()
- ВСЕГДА после изменения admin.html проверь что пара ed{Tab}()/col{Tab}() работает корректно
- ВСЕГДА обновляй PROJECT_ARCHITECTURE.md если меняешь структуру данных или добавляешь новые Firestore коллекции
- При добавлении нового поля: обнови site-data.js (defaults) + admin ed/col функции + loader.js (render)
- Коммить после каждого завершённого изменения с понятным сообщением на русском

## Частые ошибки (не повторяй)
- Забыть обновить col{Tab}() после добавления поля в ed{Tab}()
- Использовать innerHTML без экранирования
- Сломать drag&drop при изменении структуры list editor
- Забыть что fsGet() может вернуть null (Firestore недоступен)
- Менять ID полей в формах не обновив collectCurrent()

## Как сообщать об ошибках (для владельца)
Владелец не программист. Если он говорит "не работает" — спроси:
1. Какая страница / вкладка админки
2. Что он нажал / изменил
3. Что он ожидал увидеть
4. Что увидел вместо этого
